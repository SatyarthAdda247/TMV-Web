# Product Requirements Document (PRD) Summary
## TMV Studios Website

### 1. Product Overview

**Product Name:** TMV Studios Website  
**Product Type:** Corporate Portfolio Website  
**Target Audience:** Potential clients, creative industry professionals, production companies  
**Primary Goal:** Showcase TMV Studios' creative work and services in animation, live action, VFX, and CGI

**Brand Identity:**
- Tagline: "Everything you can imagine is real"
- Modern, creative, and professional aesthetic
- Green accent color (#54FF85) for brand recognition
- Clean, minimalist design approach

### 2. User Personas

**Primary Persona - Creative Director/Producer:**
- Age: 30-50
- Goal: Find a production studio for their next project
- Needs: Quick access to portfolio, service information, contact details
- Pain Points: Difficulty assessing studio capabilities, unclear pricing/process

**Secondary Persona - Marketing Manager:**
- Age: 25-45
- Goal: Explore creative solutions for brand campaigns
- Needs: Examples of previous work, service offerings, easy contact
- Pain Points: Time-consuming research, unclear specializations

**Tertiary Persona - Industry Professional:**
- Age: 20-60
- Goal: Learn about TMV Studios' original content and capabilities
- Needs: Portfolio browsing, company information, social proof

### 3. Core Features & User Stories

#### 3.1 Navigation & Discovery
**User Story:** As a visitor, I want to easily navigate the site so I can find relevant information quickly.

**Requirements:**
- Responsive navigation menu (mobile hamburger, desktop horizontal)
- Clear hierarchy: Home, Portfolio (Animation, Live Action, VFX/CGI), Originals, Services, About
- Search functionality for content discovery
- Breadcrumb navigation for deep pages
- Smooth scrolling and transitions

#### 3.2 Portfolio Showcase
**User Story:** As a potential client, I want to view TMV's work so I can assess their capabilities.

**Requirements:**
- Portfolio organized by category (Animation, Live Action Shoots, VFX and CGI)
- High-quality media display (images/videos)
- Project descriptions and details
- Filtering and sorting capabilities
- Responsive gallery layout

#### 3.3 Hero Section
**User Story:** As a first-time visitor, I want to immediately understand what TMV Studios does.

**Requirements:**
- Compelling hero image with overlay
- Clear value proposition: "Everything you can imagine is real"
- Call-to-action button: "Discover More"
- Responsive design across all devices
- Fast loading time

#### 3.4 Contact & Communication
**User Story:** As an interested client, I want to easily contact TMV Studios.

**Requirements:**
- Email: raman@themadvirus.com
- Phone: +91-844-751-4242
- LinkedIn profile link
- Contact form (if implemented)
- Footer with persistent contact information

#### 3.5 Services Information
**User Story:** As a potential client, I want to understand what services TMV offers.

**Requirements:**
- Dedicated "Our Services" page
- Clear service descriptions
- Pricing information (if applicable)
- Process overview
- Case studies or examples

#### 3.6 Original Content
**User Story:** As an industry professional, I want to explore TMV's original productions.

**Requirements:**
- Dedicated "Originals" section
- Showcase of original content
- Behind-the-scenes information
- Links to view/stream content

### 4. Design Requirements

#### 4.1 Visual Design
- **Color Palette:**
  - Primary: White (#FFFFFF)
  - Secondary: Black (#000000)
  - Accent: Bright Green (#54FF85)
  - Background: Light Gray (#F7F7F7)
  - Text: Dark Gray (#1F1F1F)

- **Typography:**
  - Headings: Oswald (400, 600, 700 weights)
  - Body: Open Sans (400, 600, 700 weights)
  - Accent: Comfortaa (400, 700 weights)
  - Sizes: Responsive scaling from 9pt to 60pt

- **Layout:**
  - Maximum content width: 1280px
  - Grid-based responsive layout
  - Consistent spacing and padding
  - Card-based content organization

#### 4.2 Branding Elements
- Logo: TMV Studios wordmark
- Favicon: Custom TMV icon
- Social media images: Consistent branding
- Loading states: Branded spinners

### 5. User Experience Requirements

#### 5.1 Performance
- Page load time: < 3 seconds
- Time to interactive: < 5 seconds
- Smooth animations: 60fps
- Optimized images: WebP format with fallbacks
- Lazy loading for below-fold content

#### 5.2 Accessibility
- WCAG 2.1 AA compliance
- Keyboard navigation support
- Screen reader compatibility
- Sufficient color contrast ratios
- Alt text for all images
- Skip navigation links
- Focus indicators

#### 5.3 Mobile Experience
- Touch-friendly interface (44px minimum touch targets)
- Responsive images and videos
- Mobile-optimized navigation
- Fast mobile load times
- Gesture support (swipe, pinch-to-zoom)

### 8. Constraints & Assumptions

#### 8.1 Technical Constraints
- Built on Google Sites platform
- Limited custom backend functionality
- Dependent on Google's infrastructure
- No direct database access

#### 8.2 Business Constraints
- Budget limitations for custom development
- Timeline for launch and updates
- Content creation resources
- Maintenance capabilities

#### 8.3 Assumptions
- Users have modern browsers (last 2 versions)
- Majority of users have broadband internet
- Users are familiar with standard web navigation
- Content will be updated regularly
